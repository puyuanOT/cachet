# Q4/Q8 Shared-GPU Cachet Canary

This appendix records the Cachet-only latency canaries used to validate the
shared GPU-prefix path before promoting a new primary table. These runs should
not be mixed with the current main table because the same-configuration
baseline rows and 16k row have not been remeasured yet.

## Configuration

| Field | Value |
| --- | --- |
| Model | `Qwen/Qwen3-4B-Instruct-2507` served as `qwen3:4b-instruct` |
| Serving engine | vLLM `0.23.0` |
| Hardware | AWS g5/A10G, `g5.8xlarge` |
| Model weights | vLLM `--quantization bitsandbytes` |
| vLLM KV cache dtype | `fp8_e5m2` |
| Cachet document KV dtype | `fp8_e5m2` generated by the current Transformers materializer |
| Cachet residency | Local disk handoff bundles, prewarmed into vLLM prefix cache |
| Prefix sharing | `--enable-prefix-caching`, static `cache_salt`, prewarm before measurement |
| Request parallelism | 8 requests in flight |
| Output length | Forced 256 tokens with `ignore_eos=true` |
| Datasets | Biography, HotpotQA, MusiQue, NIAH; one prepared row per dataset, eight repeats |

## Results

Latency values are seconds. Percentiles are computed over 32 successful
request-level measurements per context. `Answer found` is permissive substring
quality; strict exact match is also shown because forced-256 generations are
verbose and do not equal the short gold answer.

| Input context | P50 TTFT | P95 TTFT | P50 TTC (256 tokens) | P95 TTC (256 tokens) | Answer found | Exact match |
| ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| 8k | 0.389 | 0.422 | 12.730 | 12.752 | 1.00 | 0.00 |
| 32k | 0.677 | 1.122 | 19.705 | 20.000 | 1.00 | 0.00 |

Both runs had 32 successful measurements and zero request errors. Connector
telemetry recorded four `load_request` events in each run, all from prewarm,
and no Cachet KV loads during measured requests.

The server logs for both runs report `load_format=bitsandbytes`,
`quantization=bitsandbytes`, `kv_cache_dtype=fp8_e5m2`,
`enable_prefix_caching=True`, 16.32 GiB available KV-cache memory, 237,728 GPU

## Caveats

These are Cachet-arm canaries, not full main-table evidence. The primary table
still needs same-configuration baseline rows, the 16k Cachet row, and a quality
protocol that does not label forced-256 verbose generations as exact match.
