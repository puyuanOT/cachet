# PR Evidence

This folder stores machine-checkable pull-request evidence sidecars generated by
`document-kv-pr-evidence`.

Each JSON file records:

- what changed and why the slice was necessary
- the package boundaries touched by the slice
- focused and broad verification commands
- whether the Refactor skill was applied
- the GPT-5.5 review outcome and whether findings were resolved

These files are traceability artifacts for the PR workflow. They are not runtime
inputs for cache generation, materialization, serving, or Databricks benchmarks.

Use `document_kv_cache.pr_evidence.evaluate_pr_evidence_directory(...)` to
validate the saved sidecars before attaching them to a release bundle or PR
review package. Directory validation walks nested slice folders by default and
ignores valid `document_kv.pr_evidence_validation.v1` summary records, so a
validation output can live beside the sidecars it summarizes without causing the
next validation pass to fail.

From the repository root, the equivalent CLI check is:

```bash
document-kv-pr-evidence --validate-directory pr-evidence
```
